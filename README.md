# 농장 작물 배치 최적화 알고리즘 비교

> 알고리즘 수업 팀 프로젝트 — 동일한 최적화 문제를 세 가지 알고리즘으로 풀고 해의 품질과 속도를 비교한다.

---

## 문제 정의

HUFS 스마트팜 데이터를 기반으로 **1년(24반기) 동안 노지·하우스 면적을 작물에 배정해 연간 수익을 최대화**하는 최적화 문제.

### 입력 데이터

| 항목 | 내용 |
|---|---|
| 작물 | 9종 (양상추, 토마토, 대파, 오이, 고추, 시금치, 무, 배추, 옥수수) |
| 시간 단위 | 상순/하순 기준 24반기 (1반기 ≈ 15일) |
| 농지 | 노지 1200m², 하우스 300m² |
| 가격 | 월별 시장가격 변동 (노지/하우스 각각 상이) |
| 성장기간 | 작물별, 노지/하우스별 상이 (2~10반기) |

### 제약 조건

- **면적 상한/하한**: 작물당 구역 면적의 **5% 이상 25% 이하** (노지 60~300m², 하우스 15~75m²). 60/15는 단위가 아니라 하한선이며 그 사이 연속값이 모두 유효하다.
- **용량 제약**: 어떤 반기에도 노지 총 점유 ≤ 1200m², 하우스 총 점유 ≤ 300m². 작물은 수확 전까지 땅을 계속 점유하므로 파종 시기와 성장기간을 함께 고려해야 한다.
- **이진 조건**: 작물을 심으면 하한(60m²/15m²) 이상, 안 심으면 정확히 0 → 순수 LP가 아닌 **MILP**

### 목적함수

```
최대화: Σ (단위수익[i,t] × 배정면적[i,t])
단위수익 = 수확월 시장가격 / 면적비율
```

## DP를 버린 이유

초기에 DP로 구현을 시도했으나 두 가지 근본적인 문제가 발견되어 보류하였다.

**1. 불필요한 이산화**
DP 상태를 정수로 만들기 위해 60m²/15m² 슬롯 단위로 강제 이산화하였다. 이는 명세에 없는 추가 제약으로, 실제 문제는 60~300m² 사이의 연속값이 모두 유효하다. 결과적으로 해 탐색 공간이 인위적으로 좁아져 수익이 낮게 나왔다.

**2. 진짜 DP가 아닌 반기별 그리디**
코드 내부에 DP 서브루틴이 있었지만, 외부 루프는 `for t in range(24)` 형태로 반기를 순서대로 결정하고 절대 되돌리지 않는 구조였다. 이는 DP가 아니라 반기 단위 그리디이며, 전역 최적을 보장하지 못한다. 진짜 전역 DP는 24반기 × 슬롯 수의 상태 공간이 필요해 현실적으로 구현 불가능하다.

→ **MILP(LP + Branch & Bound)로 대체하여 최적 보장**

---

## 알고리즘

### MILP — 베이스라인 (최적 보장)

이 문제는 "심거나 안 심거나"라는 이진 조건이 포함되어 순수 LP가 아닌 **MILP(Mixed Integer Linear Programming)** 이다. 2단계로 풀린다.

**1단계 — LP Relaxation (Simplex 직접 구현)**

이진 조건을 무시하고 `0 ≤ x ≤ MAX`로 완화한 LP를 Tableau 방식으로 푼다.

```
max  c^T x
s.t. Ax ≤ b,  x ≥ 0

- 반기별 노지 용량: Σ x_점유 ≤ 1200  (24개 제약)
- 반기별 하우스 용량: Σ x_점유 ≤ 300  (24개 제약)
- 상한: x ≤ MAX  (변수마다 1개씩)
```

slack 변수를 추가해 초기 BFS를 잡고, 목적함수 행의 가장 음수인 열을 pivot 열로, 최소 비율 테스트로 pivot 행을 선택해 반복한다. 이 LP의 최적값이 MILP의 상한이 된다.

**2단계 — Branch & Bound**

LP 해에서 `0 < x < MIN`인 변수를 찾으면 분기한다.

```
LP해: x = 30 (위반, 0 < 30 < 60)
         /              \
   Branch 0           Branch 1
   x = 0 강제          x ≥ 60 강제
   (hi = 0)            (lo = 60)
   다시 LP 풀기        다시 LP 풀기
```

각 노드에서 LP를 다시 풀고, 또 위반이 있으면 재분기한다. 위반이 없는 노드에 도달하면 MILP 해 확정. LP 상한보다 현재 최적해가 이미 높으면 해당 노드는 탐색하지 않는다(가지치기).

이 문제에서는 LP 해 자체가 0 또는 MAX 경계값만 나와 MILP 조건을 자동으로 만족하므로 1노드에서 종료된다.

**결과**: **266.62 백만원** (수학적 최적 보장)

### Greedy — 빠른 휴리스틱

모든 유효한 (작물, 반기, 농지) 조합을 후보로 수집하고, **sort 기준으로 정렬한 순서대로** 탐욕적으로 면적을 배정한다.

```python
for (작물, 반기, 농지) in sorted_candidates:
    성장기간 동안 모든 반기의 잔여 면적 확인
    가능한 최대 면적 배정 (연속값, MIN~MAX 사이)
    이후 반기의 잔여 면적 차감
```

수익 함수가 `단위수익 × 면적`으로 선형이기 때문에 면적이 클수록 유리하다. 따라서 항상 가능한 최대 면적을 배정하는 것이 반기 내 최적이다. 그러나 앞의 결정이 뒤의 선택 가능성을 제한하므로 전역 최적을 보장하지 않는다.

**sort 기준에 따라 결과가 달라지는 이유**: 탐욕 알고리즘은 순서대로 땅을 점유하기 때문에, 어떤 조합이 먼저 선택되느냐에 따라 후보 조합이 받을 수 있는 면적이 달라진다. sort 기준 A로 결과가 253.77M일 때, 기준 B로는 다른 수익이 나올 수 있다.

**결과**: **253.77 백만원** / ~1ms

### GA — 메타휴리스틱

Greedy가 멈추는 지점을 **초기 해**로 삼아, 진화 연산으로 계속 개선하는 메타휴리스틱이다.

**염색체 표현**: 각 후보 조합에 대한 0/1 비트열 — 1이면 해당 조합을 포함, 0이면 제외.

```
[1, 0, 1, 1, 0, 0, 1, ...]
 ↑           ↑
 (토마토,3월상,노지) 포함   (양상추,7월하,하우스) 제외
```

**적합도 계산 (decode)**: 염색체에서 1인 조합만 골라, sort 순서대로 그리디 배정해 총 수익을 계산한다.

**진화 과정**:

```
1. 초기 집단 생성
   - Greedy 해 1개 (sort 기준으로 생성)
   - 그리디 해에서 20% 비트 랜덤 반전 × 29개

2. 세대 반복 (100세대)
   ① 엘리트 보존: 최우수 염색체 1개 다음 세대에 그대로 복사
   ② 토너먼트 선택: 3개 랜덤 선택 → 적합도 가장 높은 것 선택
   ③ 단일점 교차: 랜덤 위치에서 두 부모 염색체를 절반씩 교환
   ④ 비트 돌연변이: 각 비트를 5% 확률로 반전
```

Greedy가 탐욕적으로 멈춰버리는 지점을 교배와 돌연변이로 벗어나 더 높은 수익을 탐색한다.

**결과**: **264.11 백만원** / ~2,000ms (세대 100, 집단 30)

## Sort 방식 상세

Greedy와 GA 두 알고리즘 모두 **셀 2의 sort 한 줄만 수정**하면 방식이 바뀐다.

```python
# 튜플: (단위수익, 성장기간, 작물번호, 파종시기, 농지종류)
#         x[0]      x[1]      x[2]

# 방식A — 단위수익 기준 (현재)
candidates.sort(key=lambda x: (x[0], -x[1], -x[2]), reverse=True)

# 방식B — 시간당 단위수익 기준
candidates.sort(key=lambda x: (x[0] / x[1], -x[2]), reverse=True)
```

| | 방식A | 방식B |
|---|---|---|
| 1순위 | 단위수익(원/m²) 높은 순 | 단위수익 ÷ 성장기간 높은 순 |
| 의미 | 1m²당 절대 수익이 큰 작물 우선 | 같은 수익이면 땅을 빨리 비워주는 작물 우선 |
| 타이브레이커 | 성장기간 짧은 순 → 작물번호 오름 | 작물번호 오름 |

**방식A**: 단위수익이 아주 높지만 성장기간이 긴 작물(예: 토마토)을 우선 배정. 땅을 오래 묶어도 수익이 크면 먼저 선점.

**방식B**: 성장기간 대비 수익을 보기 때문에, 비슷한 수익이면 빨리 자라는 작물을 먼저 배정. 땅 회전율을 중시하는 방식.

MILP는 sort와 무관하게 항상 동일한 최적해(266.62M)를 반환한다.
